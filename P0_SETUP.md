# Подключение заявок и Метрики

Локальная версия не показывает ложный успех: подтверждение появляется только после ответа `2xx` от серверного обработчика.

## Переменные сборки

Скопируйте `.env.example` в `.env.local` и заполните:

- `VITE_LEAD_ENDPOINT` — публичный URL серверного обработчика заявок;
- `VITE_YANDEX_METRIKA_ID` — ID счётчика Метрики `110755114`. Этот ID также задан в коде как безопасное публичное значение по умолчанию.

Секреты Telegram-бота должны храниться только на сервере обработчика. Нельзя добавлять токен бота в переменные с префиксом `VITE_`: такие значения попадают в браузерный JavaScript.

Публичный обработчик развёрнут по адресу `https://leads.62.60.248.177.nip.io/api/leads`. Этот адрес задан в клиентском коде как значение по умолчанию; `VITE_LEAD_ENDPOINT` нужен только для его замены.

## Контракт обработчика заявок

Форма отправляет `POST multipart/form-data` и ожидает:

- HTTP `2xx` только после успешного сохранения или доставки заявки;
- JSON `{ "success": true }` при успехе;
- JSON `{ "success": false, "message": "..." }` и HTTP `4xx/5xx` при ошибке;
- CORS для `https://karkas-invest.ru` и локального адреса разработки.

Поля: `name`, `company`, `phone`, `service`, `details`, `drawing`, `consent`, UTM-поля, `yclid`, `page_url`, `referrer`. Поле `website` — ловушка для ботов; заполненные заявки нужно отклонять.

Обработчик должен доставлять заявку менеджеру через Telegram Bot API. Номер телефона менеджера не является адресом для метода `sendMessage`: на сервере нужны токен бота и числовой `chat_id`, а менеджер должен предварительно открыть чат с ботом и нажать Start. Телефон и Telegram-ссылка по номеру остаются на сайте как резервный ручной канал.

## Цели Метрики

Создайте JavaScript-цели с идентификаторами:

- `lead_submit_success`;
- `click_phone`;
- `click_telegram`;
- `click_whatsapp`;
- `drawing_attached`.

После согласования с клиентом выполните одну сквозную тестовую заявку с UTM-метками и проверьте получение заявки, файла и целей в Метрике. До согласования тестовые обращения не отправлять.
