# Telegram-обработчик заявок

Изолированный HTTP-сервис принимает `multipart/form-data` от сайта и передаёт заявку менеджеру через Telegram Bot API. Без токена и `chat_id` либо при `DRY_RUN=1` сервис возвращает HTTP 503 и не показывает посетителю ложный успех.

## Развёртывание

1. Создать системного пользователя `karkas-leads` без shell.
2. Скопировать `app.py` в `/opt/karkas-leads/app.py`.
3. Создать `/etc/karkas-leads.env` из примера с правами `0600`.
4. Установить unit в `/etc/systemd/system/karkas-leads.service`.
5. Выполнить `systemctl daemon-reload && systemctl enable --now karkas-leads`.
6. Установить nginx, добавить конфигурацию сайта и получить TLS-сертификат.

Unit использует `Restart=always` и `WantedBy=multi-user.target`, поэтому сервис восстанавливается после сбоя и автоматически запускается после перезагрузки сервера.

Публичный адрес текущего развёртывания: `https://leads.62.60.248.177.nip.io/api/leads`.

## Безопасная проверка

- `GET /health` проверяет процесс и конфигурацию без отправки сообщений.
- При `DRY_RUN=1` тестовый `POST /api/leads` проходит валидацию, но Telegram API не вызывается.
- Реальную сквозную заявку выполнять только после согласования с владельцем.
